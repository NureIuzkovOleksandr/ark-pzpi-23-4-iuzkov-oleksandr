ДОДАТОК Б
VISION AND SCOPE DOCUMENT (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)
1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)
1.1 Аналіз проблеми
     У сучасних приміщеннях немає централізованого контролю за температурою та вологістю, що призводить до дискомфорту, втрат енергії та складнощів у відстеженні змін клімату. Існуючі пристрої часто не взаємодіють між собою та не надають аналітики в реальному часі.
Основні проблеми:
1) Відсутність автоматичного відстеження показників.
2) Неможливість централізованого управління пристроями.
3) Обмежені можливості аналітики та оповіщень.
4) Неоптимальне використання енергії через ручне регулювання.
1.2 Аналіз аналогів
	Серед наявних рішень виділяються Nest, Netatmo, Xiaomi Smart Home. Вони дозволяють контролювати клімат, але часто не інтегруються з іншими IoT-пристроями та не забезпечують гнучких API для аналітики.
Розроблювана система надасть:
1) централізоване керування даними з усіх сенсорів,
2) автоматичне управління кліматичними пристроями,
3) веб- та мобільні інтерфейси для моніторингу та управління,
4) зберігання історії змін та аналітику в реальному часі,
5) можливість масштабування для декількох кімнат чи приміщень.



2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)
2.1 Опис призначення системи
     Система призначена для віддаленого спостереження за температурою і вологістю та управління кліматичними пристроями у кімнаті. Користувачі можуть отримувати сповіщення про відхилення показників та контролювати параметри через веб- чи мобільний додаток.
Основні цілі системи:
1) постійний моніторинг температури та вологості;
2) автоматичне регулювання кліматичних пристроїв;
3) централізоване зберігання та аналіз даних;
4) сповіщення користувача про аномалії;
5) масштабованість для різних кімнат або приміщень.
2.2 Опис бізнес-логіки
	Система забезпечує збір даних з сенсорів, обробку на сервері та управління пристроями:
1) користувачі налаштовують пороги, переглядають дані та керують пристроями;
2) IoT-модулі надсилають показники на сервер, який приймає рішення про корекцію клімату;
3) сервер формує звіти, графіки та відправляє повідомлення про відхилення.
	






3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)
3.1 Опис функціональності серверної частини та IoT-клієнта
	Сервер є центральним елементом системи, який отримує дані від сенсорів, обробляє їх та координує роботу кліматичних пристроїв. Він відповідає за логіку управління, збереження інформації та комунікацію з користувачами та IoT-пристроями.
Основні функції серверної частини:
1) Управління користувачами: реєстрація та авторизація, ведення журналу дій.
2) Обробка даних сенсорів: прийом і перевірка показників температури та вологості, фільтрація аномальних значень, збереження в базі даних.
3) Автоматичне регулювання пристроїв: на основі заданих порогів сервер відправляє команди IoT-клієнтам для включення/вимкнення кондиціонера, обігрівача або зволожувача.
4) Аналітика та звітність: формування графіків змін клімату, оповіщення користувачів про критичні показники, збереження історії подій.
5) Інтеграція з IoT-пристроями: отримання телеметрії, передача команд, моніторинг стану сенсорів та пристроїв у реальному часі.
	IoT-клієнт — це пристрій, який встановлюється у приміщенні та взаємодіє із сервером для збору даних і управління обладнанням.
Основні функції IoT-клієнта:
1) Збір показників температури та вологості.
2) Передача даних на сервер.
3) Виконання команд для зміни стану кліматичних пристроїв.
4) Виявлення технічних проблем та передача сигналів про помилки.
3.2 Опис функціональності веб-клієнта та мобільного клієнта
	Система надає два види інтерфейсів для користувачів: веб-додаток та мобільний додаток. Вони забезпечують контроль за кліматом, перегляд даних та керування пристроями.
	Основні функції веб-клієнта:
1) Перегляд поточних показників температури та вологості.
2) Налаштування порогових значень для автоматичного регулювання пристроїв.
3) Відображення графіків та історії змін клімату.
4) Управління кліматичними пристроями у приміщенні.
5) Сповіщення користувачів про перевищення критичних показників.
	Основні функції мобільного клієнта:
1) Доступ до актуальних даних з будь-якого місця.
2) Можливість отримання повідомлень про критичні зміни клімату.
3) Віддалене включення або вимкнення кліматичних пристроїв.
4) Синхронізація з сервером для оновлення стану пристроїв у реальному часі.
     
     
     
     
     
     
     
     
     
     
     
     4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)
     4.3 Опис програмних засобів та інструментів
	Для реалізації системи моніторингу клімату в приміщенні використовується сучасний технологічний стек, що забезпечує надійність, масштабованість та інтеграцію з IoT-пристроями.
	Серверна частина
1) Мова програмування: Python 3
2) Фреймворк: FastAPI — дозволяє створювати швидкий і безпечний REST API для обміну даними між веб- та мобільними клієнтами та IoT-модулями. API реалізує основні процеси системи: авторизацію користувачів, збір та обробку даних з сенсорів, управління кліматичними пристроями, аналітику та формування звітів.
3) Система керування базами даних: PostgreSQL, забезпечує надійне зберігання історії показників, підтримку транзакцій та можливість виконання складних аналітичних запитів.
4) Середовище розробки: VS Code — основне IDE для розробки серверної логіки, тестування API та налагодження.
5) Контроль версій: Git. Репозиторій розміщено на GitHub для спільної роботи, відстеження змін та історії розробки.
	Веб- та мобільні клієнти
1) Веб-клієнт:
a) Технології: React, HTML5, CSS3, JavaScript
b) Призначення: перегляд даних з сенсорів, керування кліматичними пристроями, формування графіків та звітів. 
c) Взаємодія із сервером здійснюється через REST API.
2)  Мобільний клієнт:
a) Розробка на базі React Native для кросплатформенності (Android/iOS).
b) Призначення: віддалений доступ до даних, керування пристроями та отримання повідомлень про критичні показники.
	Інтеграція з IoT-модулем
1) Апаратна платформа: Raspberry Pi. Використовується як контролер для підключення сенсорів температури та вологості, а також керування кліматичними пристроями.
2) Програмне забезпечення на IoT:
a) Мова: Python 3
b) Бібліотеки: paho-mqtt, RPi.GPIO, requests
c) Призначення: збір даних сенсорів, виконання команд від сервера та передача результатів на центральний сервер.
3) Протокол взаємодії: MQTT Забезпечує надійну публікацію та підписку для обміну даними між сервером та IoT-пристроями в режимі реального часу.
	Тестування та документація
1) Postman — для перевірки REST API та тестування сценаріїв взаємодії сервер-клієнт-IoT.
2) Swagger UI (OpenAPI) — для автоматичної генерації документації для REST API та наочна візуалізація доступних методів.

	Середовище розгортання
1) ОС сервера: Windows Server.
2) Хмарна інфраструктура: Microsoft Azure — для хостингу бази даних, серверного API та забезпечення масштабованості.
